
# User Flow

## 1. Onboarding

- User signs up with email or phone number 
- OTP verification
- Profile created
- User assigned Role = User

---

## 2. Health Wallet (Savings)

- User deposits small amounts regularly
- Balance accumulates in Health Wallet
- Funds are reserved for healthcare use
- Message reminders are sent to user on funding wallet

---

## 3. Insurance Flow (HMO-backed)

- User browses insurance plans
- User saves toward activation threshold
- Once threshold is met:
  - User subscribes
  - Enrollment sent to HMO
  - Coverage becomes active

- User can access care through partner clinics

---

## 4. Credit Flow (Lender-backed)

- User requests health credit
- Platform evaluates:
  - Savings behavior
  - Credit rules
  - Eligibility

- Admin reviews request
- Approved request is funded by lender
- Loan becomes active

- User uses funds for healthcare

---

## 5. Clinic Access

- User browses verified clinics
- User contacts or visits clinic

Payment options:
- Health Wallet
- Insurance (via HMO)
- Health Credit (funded by lender)

---

## 6. Repayment Flow

- User repays loan (via wallet or external payment)
- System updates:
  - Loan status → Repaid
  - User eligibility restored

- Repayment data improves future credit access

---

## 7. Learn & Earn (Optional Engagement Layer)

- User completes educational modules
- Earns rewards:
  - Wallet bonuses
  - Increased credit access
  - Faster insurance activation

---

## End-to-End Scenario

User saves → builds wallet →  
gets sick →  

IF insured:
→ HMO covers treatment  

IF not fully covered:
→ Credit requested → lender funds  

User receives care →  
User repays loan →  
User continues saving →  
User activates insurance over time  

---

## Platform Role in User Journey

WellPay enables access but does not:

- Provide loan capital directly
- Underwrite insurance

Instead, it connects users to:

- Lenders (for credit)
- HMOs (for insurance)
- Clinics (for care delivery)
